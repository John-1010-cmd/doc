## AI 发展简史

人工智能的发展经历了几个重要阶段，每个阶段都有其核心范式的转变。

### 从规则到统计学习

早期 AI 系统（1950s-1980s）主要依赖**符号主义**方法，通过人工编写规则来模拟智能行为。这类系统的代表包括专家系统（如 MYCIN），它们将领域知识编码为 if-then 规则。然而，规则系统面临着知识获取瓶颈——真实世界的复杂性使得手工编码所有规则变得不可能。

1980s-2000s，**统计学习**方法开始兴起。机器学习算法（如 SVM、决策树、朴素贝叶斯）能够从数据中自动学习模式，不再需要显式编码规则。这一阶段的突破包括：

- **HMM（隐马尔可夫模型）**：在语音识别领域取得重大进展
- **CRF（条件随机场）**：在序列标注任务中广泛应用
- **TF-IDF + 朴素贝叶斯**：成为文本分类的标准方案

### 深度学习的崛起

2012 年，AlexNet 在 ImageNet 竞赛中大幅领先传统方法，标志着**深度学习**时代的到来。在 NLP 领域，关键里程碑包括：

| 年份 | 事件 | 意义 |
|------|------|------|
| 2013 | Word2Vec 提出 | 分布式词表示成为标准 |
| 2014 | Seq2Seq 模型 | 端到端序列建模 |
| 2015 | Attention 机制 | 解决长序列信息丢失 |
| 2017 | Transformer 提出 | 自注意力范式革命 |
| 2018 | BERT / GPT 发布 | 预训练语言模型时代 |
| 2020 | GPT-3 发布 | 大语言模型展现出涌现能力 |
| 2023 | GPT-4 发布 | 多模态大模型 |

深度学习之所以能够超越传统方法，核心在于三点：**大规模数据**的可用性、**GPU 算力**的指数级增长、以及**更优的网络架构**设计。

## Transformer 架构核心

Transformer 是现代大语言模型的基石。2017 年 Google 在论文《Attention Is All You Need》中提出，彻底改变了 NLP 的技术范式。

### Self-Attention 机制

Self-Attention 是 Transformer 的核心创新。其基本思想是：让序列中的每个位置都能直接关注到序列中的所有其他位置，从而捕获长距离依赖关系。

#### 计算过程

Self-Attention 的计算分为以下步骤：

1. **线性变换**：将输入向量通过三个权重矩阵变换为 Query（Q）、Key（K）、Value（V）

```python
import torch
import torch.nn as nn
import math

class SelfAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        self.d_k = d_model // n_heads
        self.n_heads = n_heads
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, x, mask=None):
        B, T, C = x.shape
        # 线性变换得到 Q, K, V
        Q = self.W_q(x).view(B, T, self.n_heads, self.d_k).transpose(1, 2)
        K = self.W_k(x).view(B, T, self.n_heads, self.d_k).transpose(1, 2)
        V = self.W_v(x).view(B, T, self.n_heads, self.d_k).transpose(1, 2)

        # 计算注意力分数
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        attn = torch.softmax(scores, dim=-1)

        # 加权求和
        out = torch.matmul(attn, V)
        out = out.transpose(1, 2).contiguous().view(B, T, -1)
        return self.W_o(out)
```

2. **计算注意力分数**：Q 和 K 的点积除以缩放因子
3. **Softmax 归一化**：得到注意力权重分布
4. **加权求和**：用注意力权重对 V 进行加权求和

Self-Attention 的时间复杂度为 O(n^2)，其中 n 是序列长度。这是 Transformer 处理超长序列时的主要瓶颈。

### Multi-Head Attention

Multi-Head Attention 将 Self-Attention 扩展为多个并行的"头"，每个头学习关注不同的子空间信息：

- 不同的头可以关注**不同的语义层面**（如语法关系、语义关系、共指关系）
- 将 d_model 维度拆分为 h 个头，每个头维度为 d_model / h
- 多头的输出拼接后再经过线性变换

```
MultiHead(Q, K, V) = Concat(head_1, ..., head_h) * W_O
其中 head_i = Attention(Q * W_Q_i, K * W_K_i, V * W_V_i)
```

### 位置编码

由于 Self-Attention 本身是**置换不变**的（即不感知位置信息），Transformer 需要显式注入位置信息。原始论文使用正弦/余弦位置编码：

```python
class PositionalEncoding(nn.Module):
    def __init__(self, d_model, max_len=5000):
        super().__init__()
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len).unsqueeze(1).float()
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model)
        )
        pe[:, 0::2] = torch.sin(position * div_term)  # 偶数维度
        pe[:, 1::2] = torch.cos(position * div_term)  # 奇数维度
        self.register_buffer('pe', pe.unsqueeze(0))

    def forward(self, x):
        return x + self.pe[:, :x.size(1)]
```

后续发展出的位置编码方案包括：

- **可学习位置编码**：将位置编码作为可训练参数（BERT 使用）
- **RoPE（旋转位置编码）**：通过旋转矩阵编码相对位置（LLaMA 使用）
- **ALiBi**：在注意力分数上添加距离相关的线性偏置

## GPT 系列演进

GPT（Generative Pre-trained Transformer）系列是 OpenAI 推出的自回归语言模型，其演进历程体现了大模型的核心技术趋势。

### GPT-1：开创性验证

2018 年发布的 GPT-1 验证了"**生成式预训练 + 判别式微调**"范式的有效性：

- 参数量：1.17 亿
- 架构：12 层 Transformer Decoder
- 预训练：在大规模无标注文本上进行语言建模
- 微调：在下游任务的有标注数据上微调

### GPT-2：规模扩展与 Zero-shot

2019 年的 GPT-2 展示了**规模扩展**带来的能力提升：

- 参数量：15 亿（最大版本）
- 核心改进：更大的模型、更多的数据、去掉了微调阶段
- 关键发现：模型规模增大后展现出 Zero-shot 学习能力

### GPT-3：涌现能力

2020 年的 GPT-3 是一个里程碑，参数量达到 **1750 亿**：

- 核心突破：展现出**涌现能力**（Emergent Abilities），即小模型不具备但大模型突然展现的能力
- 少样本学习：通过 In-context Learning，无需梯度更新即可完成新任务
- 技术细节：使用了稀疏注意力（Sparse Attention）来处理更长序列

### GPT-4：多模态与推理

2023 年发布的 GPT-4 进一步扩展了能力边界：

- 支持多模态输入（图像 + 文本）
- 推理能力大幅提升，在专业考试中达到人类水平
- 引入 RLHF（基于人类反馈的强化学习）对齐技术
- 架构细节未公开，但推测采用了 MoE（Mixture of Experts）结构

## Tokenization

Tokenization 是将原始文本转换为模型输入的第一步，直接影响模型的词汇量和处理效率。

### BPE（Byte-Pair Encoding）

BPE 是目前最常用的 Tokenization 方法，GPT 系列均使用此方案：

```
# BPE 算法流程
初始词汇表 = 所有单个字符
重复以下步骤直到达到目标词汇量：
    1. 统计当前语料中所有相邻 token 对的频率
    2. 合并频率最高的 token 对，生成新 token
    3. 更新词汇表和语料中的 token 表示
```

BPE 的优势在于：
- 能自动平衡词级和字符级表示
- 可以处理未登录词（OOV）
- 常见词有较短的编码，罕见词自动拆分为子词

### WordPiece

WordPiece 与 BPE 类似，但选择合并对的标准不同：

- BPE 选择**频率最高**的相邻对
- WordPiece 选择使**语言模型似然增益最大**的相邻对
- BERT 使用 WordPiece 作为 Tokenizer

### SentencePiece

SentencePiece 是一个语言无关的 Tokenization 框架：

- 将文本视为原始字节流，不依赖预分词
- 同时支持 BPE 和 Unigram 两种算法
- LLaMA、T5 等模型使用 SentencePiece

## 上下文窗口与上下文学习

### 上下文窗口

上下文窗口（Context Window）是模型一次能处理的最大 token 数量：

| 模型 | 上下文长度 |
|------|-----------|
| GPT-3 | 4K |
| GPT-4 Turbo | 128K |
| Claude 3 | 200K |
| Gemini 1.5 Pro | 1M |
| LLaMA 3.1 | 128K |

扩展上下文窗口面临的技术挑战：
- 注意力计算的二次复杂度
- 位置编码的外推能力
- 长文本中的信息遗忘问题

### 上下文学习（In-Context Learning, ICL）

ICL 是大模型的核心能力之一，指模型通过 Prompt 中的示例来学习新任务，**无需更新模型参数**：

```python
# ICL 示例：Few-shot 情感分类
prompt = """
判断以下评论的情感倾向（正面/负面）：

评论：这个产品质量很好，非常满意！
情感：正面

评论：物流太慢了，等了一周才到。
情感：负面

评论：性价比很高，推荐购买。
情感：
"""
# 模型输出：正面
```

ICL 的工作原理尚在研究中，但主流假说认为：
- 预训练数据中包含大量相似的任务模式
- Transformer 的注意力机制隐式实现了梯度下降
- ICL 的效果随模型规模增大而提升

## 微调技术

微调是将预训练模型适配到特定任务的关键步骤。

### Full Fine-tuning

全量微调更新模型的所有参数：

```python
from transformers import AutoModelForCausalLM, Trainer, TrainingArguments

model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-2-7b")

# 全量微调：所有参数都参与梯度更新
training_args = TrainingArguments(
    output_dir="./output",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    learning_rate=2e-5,
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=train_dataset,
)
trainer.train()
```

缺点：需要大量 GPU 显存（7B 模型全量微调约需 60GB+），容易过拟合。

### LoRA（Low-Rank Adaptation）

LoRA 通过在权重矩阵旁添加低秩分解矩阵来实现参数高效的微调：

```
原始权重更新：W = W_0 + delta_W
LoRA 分解：  delta_W = A * B
其中 A in R^{d x r}, B in R^{r x d}, r << d
```

```python
from peft import LoraConfig, get_peft_model

config = LoraConfig(
    r=16,                    # 低秩维度
    lora_alpha=32,           # 缩放因子
    target_modules=["q_proj", "v_proj"],  # 应用 LoRA 的模块
    lora_dropout=0.05,
    task_type="CAUSAL_LM",
)

model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-2-7b")
model = get_peft_model(model, config)
# 可训练参数仅占总参数的 0.1%~1%
model.print_trainable_parameters()
```

### QLoRA

QLoRA 在 LoRA 基础上引入量化技术，进一步降低显存需求：

- **4-bit NormalFloat 量化**：一种针对正态分布优化的数据类型
- **双重量化**：对量化常数本身再量化，进一步节省显存
- **分页优化器**：利用 CPU 内存处理显存峰值

QLoRA 使得在单张 48GB GPU 上微调 65B 参数模型成为可能。

## 推理优化

大模型的推理成本是生产部署的核心挑战。

### KV Cache

KV Cache 是自回归推理的关键优化：

- **原理**：缓存已计算 token 的 Key 和 Value 矩阵，避免重复计算
- **代价**：显存占用随序列长度线性增长
- **优化方向**：PagedAttention（vLLM）、GQA（Grouped Query Attention）、MQA（Multi-Query Attention）

```
# 无 KV Cache：每步需要重新计算所有 token 的注意力
# 有 KV Cache：每步只计算新 token，复用之前的 K、V
Step 1: compute K1, V1 -> output
Step 2: reuse K1, V1; compute K2, V2 -> output
Step 3: reuse K1-2, V1-2; compute K3, V3 -> output
```

### 量化

量化将模型权重从高精度表示压缩为低精度表示：

| 方法 | 精度 | 显存节省 | 质量损失 |
|------|------|---------|---------|
| FP16 | 16-bit | 基线 | 无 |
| INT8 | 8-bit | ~50% | 极小 |
| INT4 (GPTQ) | 4-bit | ~75% | 小 |
| INT4 (GGUF) | 4-bit | ~75% | 小 |

```python
# 使用 bitsandbytes 进行量化
from transformers import BitsAndBytesConfig

quantization_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_compute_dtype=torch.float16,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_use_double_quant=True,
)

model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b",
    quantization_config=quantization_config,
)
```

### 投机采样

投机采样（Speculative Decoding）通过"小模型猜测 + 大模型验证"来加速推理：

1. 小模型（Draft Model）快速生成 k 个候选 token
2. 大模型（Target Model）并行验证这些 token
3. 接受正确的 token，拒绝错误的 token 后重新生成

这种方法可以在不损失输出质量的前提下实现 2-3 倍的推理加速。

## 如何评估大模型

### 基准测试

常用的自动评估基准：

- **MMLU**（Massive Multitask Language Understanding）：涵盖 57 个学科的知识问答
- **HumanEval**：代码生成能力评估
- **GSM8K**：小学数学推理
- **C-Eval**：中文多学科评估
- **HELM**（Holistic Evaluation of Language Models）：全方位评估框架

### 人工评估

自动基准存在局限性（数据污染、与实际需求脱节），人工评估仍然不可替代：

- **对齐评估**：模型输出是否符合人类期望（有用性、诚实性、无害性）
- **偏好评估**：人类标注员对模型输出进行排序
- **红队测试**：专门寻找模型的安全漏洞和边界情况

### 评估最佳实践

1. 使用多个基准测试，避免单一指标片面性
2. 关注任务相关的领域评估，而非通用排名
3. 定期检查基准数据是否被污染
4. 结合自动评估和人工评估
5. 在真实业务场景中建立 A/B 测试管道
