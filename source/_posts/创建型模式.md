创建型设计模式关注对象的创建机制，提供灵活的对象创建方式，将对象的创建与使用分离。本文深入分析五种创建型模式，结合 Spring 源码与真实业务场景，帮助开发者在实战中正确运用。

## 单例模式

### 概念

单例模式确保一个类只有一个实例，并提供全局访问点。适用于需要共享状态或控制资源的场景，如数据库连接池、配置管理器、日志对象等。

### UML 类图

```
+-------------------+
|    Singleton      |
+-------------------+
| - instance: Singleton |
+-------------------+
| + getInstance(): Singleton |
| - Singleton()          |
+-------------------+
```

核心要素：私有构造方法、静态持有自身实例、公开静态获取方法。

### 代码示例

#### 饿汉式

饿汉式在类加载时就完成初始化，线程安全但无法延迟加载。

```java
public class EagerSingleton {
    // 类加载时即创建实例
    private static final EagerSingleton INSTANCE = new EagerSingleton();

    private EagerSingleton() {
        // 防止反射攻击
        if (INSTANCE != null) {
            throw new IllegalStateException("单例已存在");
        }
    }

    public static EagerSingleton getInstance() {
        return INSTANCE;
    }
}
```

**优点**：实现简单，线程安全（由 ClassLoader 保证）。
**缺点**：无法延迟加载，若实例未使用则浪费资源。

#### 懒汉式

懒汉式在首次使用时才创建实例，支持延迟加载。

```java
public class LazySingleton {
    private static volatile LazySingleton instance;

    private LazySingleton() {}

    public static LazySingleton getInstance() {
        if (instance == null) {
            synchronized (LazySingleton.class) {
                if (instance == null) {
                    instance = new LazySingleton();
                }
            }
        }
        return instance;
    }
}
```

#### DCL（Double-Checked Locking）双检锁

DCL 是懒汉式的高性能版本，避免每次获取实例都加锁。

```java
public class DCLSingleton {
    // volatile 防止指令重排序
    private static volatile DCLSingleton instance;

    private DCLSingleton() {}

    public static DCLSingleton getInstance() {
        // 第一次检查，避免不必要的同步
        if (instance == null) {
            synchronized (DCLSingleton.class) {
                // 第二次检查，防止重复创建
                if (instance == null) {
                    instance = new DCLSingleton();
                }
            }
        }
        return instance;
    }
}
```

**关键点**：`volatile` 关键字不可或缺。对象创建分为三步：分配内存 -> 初始化对象 -> 引用指向内存地址。若无 `volatile`，JIT 可能将步骤 2 和 3 重排序，导致其他线程拿到未初始化完成的对象。

#### 枚举实现

Effective Java 推荐的最佳实践，天生防反射、防序列化。

```java
public enum EnumSingleton {
    INSTANCE;

    private AtomicLong counter = new AtomicLong(0);

    public long nextId() {
        return counter.incrementAndGet();
    }
}

// 使用方式
long id = EnumSingleton.INSTANCE.nextId();
```

**为什么枚举是最好的单例实现**：
1. 线程安全由 JVM 保证（枚举实例在类加载时创建）
2. 天然防止反射创建新实例（`Constructor#newInstance` 对枚举有特殊检查）
3. 天然防止反序列化创建新实例（枚举的 `readResolve` 由 JVM 实现）

### Spring 中的应用

Spring Bean 默认作用域就是单例。Spring 的单例实现与传统的单例模式有本质区别：

```java
// Spring DefaultSingletonBeanRegistry 核心代码
public class DefaultSingletonBeanRegistry {
    // 一级缓存：存放完全初始化好的 Bean
    private final Map<String, Object> singletonObjects = new ConcurrentHashMap<>(256);
    // 二级缓存：存放早期暴露的 Bean（未完成属性注入）
    private final Map<String, Object> earlySingletonObjects = new ConcurrentHashMap<>(16);
    // 三级缓存：存放 Bean 工厂（用于生成早期引用）
    private final Map<String, ObjectFactory<?>> singletonFactories = new HashMap<>(16);

    protected Object getSingleton(String beanName, boolean allowEarlyReference) {
        Object singletonObject = this.singletonObjects.get(beanName);
        if (singletonObject == null && isSingletonCurrentlyInCreation(beanName)) {
            singletonObject = this.earlySingletonObjects.get(beanName);
            if (singletonObject == null && allowEarlyReference) {
                synchronized (this.singletonObjects) {
                    singletonObject = this.singletonObjects.get(beanName);
                    if (singletonObject == null) {
                        singletonObject = this.earlySingletonObjects.get(beanName);
                        if (singletonObject == null) {
                            ObjectFactory<?> singletonFactory = this.singletonFactories.get(beanName);
                            if (singletonFactory != null) {
                                singletonObject = singletonFactory.getObject();
                                this.earlySingletonObjects.put(beanName, singletonObject);
                                this.singletonFactories.remove(beanName);
                            }
                        }
                    }
                }
            }
        }
        return singletonObject;
    }
}
```

**Spring 三级缓存解决循环依赖**：
- `singletonObjects`：完整 Bean
- `earlySingletonObjects`：早期引用（可能被 AOP 代理）
- `singletonFactories`：ObjectFactory，按需创建早期引用

Spring 的单例是**注册式单例**（容器管理），而非经典的**自管理单例**，每个 IoC 容器内唯一。

## 工厂方法模式

### 概念

工厂方法模式定义一个创建对象的接口，让子类决定实例化哪个类。将对象创建延迟到子类，实现创建与使用的解耦。

### UML 类图

```
+------------------+        +------------------+
|   Product        |        |   Creator        |
+------------------+        +------------------+
| + operation()    |        | + factoryMethod()|
+------------------+        | + doSomething()  |
        ^                   +------------------+
        |                           ^
        |                           |
+------------------+        +------------------+
| ConcreteProduct  |        | ConcreteCreator  |
+------------------+        +------------------+
| + operation()    |        | + factoryMethod()|
+------------------+        +------------------+
```

### 代码示例

```java
// 抽象产品
public interface Logger {
    void log(String message);
}

// 具体产品
public class FileLogger implements Logger {
    @Override
    public void log(String message) {
        System.out.println("File: " + message);
    }
}

public class DatabaseLogger implements Logger {
    @Override
    public void log(String message) {
        System.out.println("DB: " + message);
    }
}

// 抽象工厂
public abstract class LoggerFactory {
    public abstract Logger createLogger();

    // 工厂也可以包含通用业务逻辑
    public void log(String message) {
        Logger logger = createLogger();
        logger.log(message);
    }
}

// 具体工厂
public class FileLoggerFactory extends LoggerFactory {
    @Override
    public Logger createLogger() {
        return new FileLogger();
    }
}
```

### Spring 中的应用

#### FactoryBean

Spring 的 `FactoryBean` 是工厂方法模式的经典实现，允许自定义 Bean 的创建逻辑。

```java
public class MyFactoryBean implements FactoryBean<MyService> {
    private String serviceType;

    public void setServiceType(String serviceType) {
        this.serviceType = serviceType;
    }

    @Override
    public MyService getObject() {
        if ("remote".equals(serviceType)) {
            return new RemoteMyService();
        }
        return new LocalMyService();
    }

    @Override
    public Class<?> getObjectType() {
        return MyService.class;
    }

    @Override
    public boolean isSingleton() {
        return true;
    }
}
```

MyBatis 的 `SqlSessionFactoryBean` 就是典型的 FactoryBean 实现：

```java
public class SqlSessionFactoryBean implements FactoryBean<SqlSessionFactory>, InitializingBean {
    private Resource configLocation;
    private DataSource dataSource;

    @Override
    public SqlSessionFactory getObject() throws Exception {
        return buildSqlSessionFactory();
    }

    @Override
    public Class<? extends SqlSessionFactory> getObjectType() {
        return SqlSessionFactory.class;
    }
}
```

#### BeanFactory

`BeanFactory` 是 Spring IoC 容器的根接口，本身就是一个大工厂：

```java
public interface BeanFactory {
    Object getBean(String name) throws BeansException;
    <T> T getBean(Class<T> requiredType) throws BeansException;
    <T> T getBean(String name, Class<T> requiredType) throws BeansException;
    boolean containsBean(String name);
    boolean isSingleton(String name);
}
```

`BeanFactory` 负责创建、管理、装配 Bean，是 Spring 框架最核心的工厂。

## 抽象工厂模式

### 概念

抽象工厂提供一个创建一系列相关或相互依赖对象的接口，而无需指定它们的具体类。适用于需要创建一整个产品族的场景。

### UML 类图

```
+-------------------+         +-------------------+
| AbstractFactory   |         | AbstractProductA  |
+-------------------+         +-------------------+
| + createProductA()|-------->| + operationA()    |
| + createProductB()|         +-------------------+
+-------------------+                  ^
        ^                              |
        |                     +-------------------+
+-------------------+         | ProductA1         |
| ConcreteFactory1  |         | ProductA2         |
+-------------------+         +-------------------+
| + createProductA()|
| + createProductB()|
+-------------------+
```

### 代码示例：多数据库连接工厂

```java
// 抽象产品：连接
public interface DbConnection {
    void connect();
    void disconnect();
}

// 抽象产品：命令
public interface DbCommand {
    int execute(String sql);
    ResultSet query(String sql);
}

// 抽象工厂
public interface DatabaseFactory {
    DbConnection createConnection();
    DbCommand createCommand();
}

// MySQL 产品族
public class MySqlConnection implements DbConnection {
    @Override
    public void connect() {
        System.out.println("连接 MySQL 数据库");
    }
    @Override
    public void disconnect() {
        System.out.println("断开 MySQL 连接");
    }
}

public class MySqlCommand implements DbCommand {
    @Override
    public int execute(String sql) {
        System.out.println("MySQL 执行: " + sql);
        return 0;
    }
    @Override
    public ResultSet query(String sql) {
        System.out.println("MySQL 查询: " + sql);
        return null;
    }
}

// MySQL 工厂
public class MySqlFactory implements DatabaseFactory {
    @Override
    public DbConnection createConnection() {
        return new MySqlConnection();
    }
    @Override
    public DbCommand createCommand() {
        return new MySqlCommand();
    }
}

// PostgreSQL 工厂
public class PostgresFactory implements DatabaseFactory {
    @Override
    public DbConnection createConnection() {
        return new PostgresConnection();
    }
    @Override
    public DbCommand createCommand() {
        return new PostgresCommand();
    }
}
```

### Spring 中的应用

Spring 的 `ApplicationContext` 本质上就是抽象工厂模式的体现。不同环境的 `ApplicationContext` 实现会创建不同的 Bean 集合：

```java
// 不同环境使用不同的 ApplicationContext（工厂）
// XML 配置
ApplicationContext xmlContext = new ClassPathXmlApplicationContext("beans.xml");

// 注解配置
ApplicationContext annotationContext = new AnnotationConfigApplicationContext(AppConfig.class);

// Web 环境
ApplicationContext webContext = new AnnotationConfigWebApplicationContext();
```

## 建造者模式

### 概念

建造者模式将复杂对象的构建与表示分离，使同样的构建过程可以创建不同的表示。特别适用于构造参数多、可选参数多的场景。

### UML 类图

```
+------------------+     +------------------+     +------------------+
|   Director       |     |   Builder        |     |   Product        |
+------------------+     +------------------+     +------------------+
| + construct()    |---->| + buildPartA()   |     | - fieldA         |
+------------------+     | + buildPartB()   |     | - fieldB         |
                         | + getResult()    |     +------------------+
                         +------------------+
```

### 代码示例

#### 经典建造者

```java
// 产品类
public class HttpRequest {
    private final String url;
    private final String method;
    private final Map<String, String> headers;
    private final String body;
    private final int timeout;

    private HttpRequest(Builder builder) {
        this.url = builder.url;
        this.method = builder.method;
        this.headers = Collections.unmodifiableMap(builder.headers);
        this.body = builder.body;
        this.timeout = builder.timeout;
    }

    public static class Builder {
        private final String url;
        private String method = "GET";
        private Map<String, String> headers = new HashMap<>();
        private String body;
        private int timeout = 30000;

        public Builder(String url) {
            this.url = Objects.requireNonNull(url, "URL 不能为空");
        }

        public Builder method(String method) {
            this.method = method;
            return this;
        }

        public Builder header(String key, String value) {
            this.headers.put(key, value);
            return this;
        }

        public Builder body(String body) {
            this.body = body;
            return this;
        }

        public Builder timeout(int timeout) {
            this.timeout = timeout;
            return this;
        }

        public HttpRequest build() {
            return new HttpRequest(this);
        }
    }
}

// 使用方式
HttpRequest request = new HttpRequest.Builder("https://api.example.com/users")
        .method("POST")
        .header("Content-Type", "application/json")
        .body("{\"name\": \"张三\"}")
        .timeout(5000)
        .build();
```

#### Lombok @Builder

Lombok 的 `@Builder` 注解自动生成建造者代码：

```java
@Builder
public class UserDTO {
    @Builder.Default
    private String name = "匿名";

    private Integer age;
    private String email;
    private List<String> roles;
}

// 使用方式
UserDTO user = UserDTO.builder()
        .name("张三")
        .age(25)
        .email("zhangsan@example.com")
        .roles(Arrays.asList("ADMIN", "USER"))
        .build();
```

### Spring 中的应用

#### Spring Security Builder

Spring Security 大量使用建造者模式，最典型的是 `HttpSecurity` 配置：

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable())
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/public/**").permitAll()
                .requestMatchers("/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            )
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            )
            .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }
}
```

#### StringBuilder

Java 标准库中的 `StringBuilder` 也是建造者模式的体现，将多个字符串逐步拼接成最终结果：

```java
String result = new StringBuilder()
        .append("SELECT u.name, u.age ")
        .append("FROM users u ")
        .append("WHERE u.age > 18 ")
        .append("ORDER BY u.name")
        .toString();
```

## 原型模式

### 概念

原型模式通过复制已有实例来创建新对象，而非通过 `new` 关键字。适用于创建成本高或构造复杂的对象。

### UML 类图

```
+-------------------+
|   Prototype       |
+-------------------+
| + clone(): Prototype |
+-------------------+
        ^
        |
+-------------------+
| ConcretePrototype |
+-------------------+
| - field: String   |
| + clone()         |
+-------------------+
```

### 代码示例

#### 浅拷贝

```java
public class ShallowCopy implements Cloneable {
    private String name;
    private Address address; // 引用类型，浅拷贝时只复制引用

    @Override
    public ShallowCopy clone() {
        try {
            return (ShallowCopy) super.clone();
        } catch (CloneNotSupportedException e) {
            throw new AssertionError();
        }
    }
}

public class Address {
    private String city;
    private String street;
    // getter/setter...
}
```

**浅拷贝的问题**：修改克隆对象的引用类型字段会影响原对象。

#### 深拷贝

```java
// 方式1：逐层 clone
public class DeepCopy implements Cloneable {
    private String name;
    private Address address;

    @Override
    public DeepCopy clone() {
        try {
            DeepCopy copy = (DeepCopy) super.clone();
            copy.address = address.clone(); // 递归克隆引用类型
            return copy;
        } catch (CloneNotSupportedException e) {
            throw new AssertionError();
        }
    }
}

// 方式2：序列化（推荐）
public class SerializableDeepCopy implements Serializable {
    private String name;
    private Address address;

    @SuppressWarnings("unchecked")
    public <T> T deepCopy() {
        try {
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            ObjectOutputStream oos = new ObjectOutputStream(bos);
            oos.writeObject(this);

            ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
            ObjectInputStream ois = new ObjectInputStream(bis);
            return (T) ois.readObject();
        } catch (Exception e) {
            throw new RuntimeException("深拷贝失败", e);
        }
    }
}
```

#### 深拷贝 vs 浅拷贝对比

| 特性 | 浅拷贝 | 深拷贝 |
|------|--------|--------|
| 基本类型 | 复制值 | 复制值 |
| 引用类型 | 复制引用（共享） | 复制对象（独立） |
| 实现方式 | `Object.clone()` | 序列化 / 逐层 clone |
| 性能 | 高 | 较低 |

### Spring 中的应用

#### Bean 的 Prototype Scope

Spring 中将 Bean 的作用域设为 `prototype`，每次获取 Bean 时都创建新实例：

```java
@Component
@Scope("prototype")
public class PrototypeBean {
    private String state;

    public void setState(String state) {
        this.state = state;
    }
}

@Service
public class BusinessService {
    @Autowired
    private ApplicationContext context;

    public void process() {
        // 每次调用都获取新实例（类似原型模式）
        PrototypeBean bean1 = context.getBean(PrototypeBean.class);
        bean1.setState("state-1");

        PrototypeBean bean2 = context.getBean(PrototypeBean.class);
        bean2.setState("state-2");
        // bean1 和 bean2 是不同实例
    }
}
```

**注意事项**：在 Singleton Bean 中注入 Prototype Bean 时，需使用 `@Lookup` 方法注入或 `ObjectFactory`/`Provider` 来确保每次获取新实例：

```java
@Service
public class SingletonService {
    // 方式1：@Lookup 方法注入
    @Lookup
    public abstract PrototypeBean getPrototypeBean();

    // 方式2：ObjectFactory
    @Autowired
    private ObjectFactory<PrototypeBean> prototypeBeanFactory;

    public void execute() {
        PrototypeBean bean = prototypeBeanFactory.getObject();
    }
}
```

## 总结

| 模式 | 核心意图 | 典型场景 | Spring 应用 |
|------|----------|----------|-------------|
| 单例 | 保证唯一实例 | 配置管理、连接池 | Bean 默认 Scope |
| 工厂方法 | 子类决定创建哪个类 | 框架扩展点 | FactoryBean |
| 抽象工厂 | 创建产品族 | 多数据库适配 | ApplicationContext |
| 建造者 | 分步构建复杂对象 | 多参数构造 | Security Builder |
| 原型 | 复制已有实例 | 创建成本高的对象 | Prototype Scope |

创建型模式的核心思想是**将对象的创建过程封装起来**，让客户端代码不依赖具体类的构造方法，从而降低耦合度。在实际开发中，不必刻意使用每种模式，但当场景匹配时，正确运用创建型模式能显著提升代码的可维护性和扩展性。
